
# 📘 Лабораторная работа №1

## Вариант 10

### Тема: gRPC-сервис TaskManager (Server Streaming RPC)

---

## 📌 Задание

Реализовать gRPC-сервис **TaskManager** с методом:

```
GetTasksForUser(UserID)
```

Метод должен возвращать **поток задач** для конкретного пользователя
(**Server streaming RPC**).

Дополнительно реализован метод:

```
UpdateTaskStatus
```

для изменения статуса выполнения задачи.

---
## 🏗 Архитектура решения

В работе реализована классическая клиент-серверная архитектура.

+-------------+        gRPC         +-------------+
|   Клиент    |  <--------------->  |   Сервер    |
|  client.py  |                     |  server.py  |
+-------------+                     +-------------+
                                          |
                                          |
                                   +---------------+
                                   |   tasks_db    |
                                   | (в памяти)    |
                                   +---------------+
Компоненты системы:

Client (client.py)
Отправляет запросы серверу и получает ответы.

Server (server.py)
Обрабатывает запросы клиента и возвращает данные.

task.proto
Контракт взаимодействия между клиентом и сервером.

tasks_db
Имитация базы данных (список задач в памяти).

---

## 🧩 Описание реализации

В рамках лабораторной работы был создан gRPC-сервис **TaskManager**.

Описание сервиса выполнено в файле:

```
task.proto
```

Сервис содержит два метода:

* `GetTasksForUser` — возвращает поток задач пользователя;
* `UpdateTaskStatus` — изменяет статус выполнения задачи.

Метод `GetTasksForUser` использует ключевое слово `stream`, что позволяет серверу отправлять несколько сообщений в ответ на один запрос клиента.

## ⚙ Генерация gRPC-кода

После создания `task.proto` была выполнена команда:

```bash
python -m grpc_tools.protoc -I. --python_out=. --grpc_python_out=. task.proto
```

В результате были автоматически созданы файлы:

* `task_pb2.py`
* `task_pb2_grpc.py`

Эти файлы обеспечивают взаимодействие клиента и сервера через gRPC.


---

## 🖥 Реализация сервера

Серверная часть реализована в файле:

```
server.py
```

Создан класс `TaskManager`, наследующийся от `TaskManagerServicer`.

Реализованы методы:

* `GetTasksForUser` — возвращает задачи с использованием `yield`;
* `UpdateTaskStatus` — изменяет статус задачи в памяти (имитация базы данных).

Сервер запускается на порту:

```
50051
```

и ожидает подключения клиентов.

## 💻 Реализация клиента

Клиентская часть реализована в файле:

```
client.py
```

Подключение выполняется через:

```python
grpc.insecure_channel('localhost:50051')
```

Клиент позволяет:

* получить список задач пользователя;
* изменить статус выполнения задачи.

Получение задач реализовано через цикл `for`, который обрабатывает поток ответов сервера.
