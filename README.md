# distributed_systems
Лабораторная работа №1

Номер варианта: 10

Задание

Реализовать gRPC-сервис TaskManager с методом
GetTasksForUser(UserID) для получения потока задач конкретного пользователя
(Server streaming RPC).

Дополнительно реализовать метод изменения статуса выполнения задачи.

Описание реализации

В рамках лабораторной работы был создан gRPC-сервис TaskManager.
Описание сервиса выполнено в файле task.proto с использованием языка Protocol Buffers.

В сервисе определены два метода:

GetTasksForUser — возвращает поток задач пользователя (server streaming RPC);

UpdateTaskStatus — изменяет статус выполнения задачи.

Метод GetTasksForUser использует ключевое слово stream, что позволяет серверу отправлять несколько сообщений в ответ на один запрос клиента.
Таким образом реализуется серверный поток данных.

Эти файлы обеспечивают взаимодействие клиента и сервера через gRPC.

Серверная часть реализована в файле server.py.
Создан класс TaskManager, который наследуется от TaskManagerServicer.

Метод GetTasksForUser возвращает задачи с использованием оператора yield, что позволяет отправлять данные клиенту по одной записи за раз.

Метод UpdateTaskStatus изменяет статус задачи в памяти (используется имитация базы данных в виде списка словарей).

Сервер запускается на порту 50051 и ожидает подключения клиентов.

Клиентская часть реализована в файле client.py.
Клиент подключается к серверу через:

grpc.insecure_channel('localhost:50051')

Клиент предоставляет возможность:

получить список задач пользователя;

изменить статус выполнения задачи.

Получение задач реализовано через цикл for, который обрабатывает поток сообщений от сервера.
